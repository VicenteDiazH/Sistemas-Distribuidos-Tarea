Para iniciar docker se deben tener 4 archivos principales:
git clone https://github.com/VicenteDiazH/Sistemas-Distribuidos-Tarea.git

Archivos:
docker-compose.yml
Dockerfile
index.js
package.json

(Levantamiento lento)
En caso de que sea necesario reconstruir las imagenes de cada servicio
ya que hubo un cambio en el Dockerfile o cambiaste las dependencias o
quieres asegurarte de que la imagen este actualizada, utilizar:

docker-compose up --build

(Levantamiento rápido)
En caso de que modifiques el código fuente y estas desarrollando, lo
mejor es levantar contenedores ya creadas en tu máquina, no recompila
gracias a los volumenes, utilizar:

docker-compose up

=====================================================================
Para visualizar si los dockers están funcionando comprobarlo con:
docker ps
=====================================================================

Cada vez que hagan cambios:
git add .				# . para agregar todos los cambios, si no nombre en especifico del archivo
git status				# para confirmar que se agregó
git commit -m "Descripción del cambio"	# preparación para subirlo
git push -u origin master		# subir todos los cambios (master = main branch)
